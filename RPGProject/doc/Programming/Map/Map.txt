名前空間 : RPGProject.GamePlay.Map
クラス名 : Map
概要 : キャラクターが移動できるマップを表現します。
メンバー : 
int xSize : マップのxサイズ
int ySize : マップのyサイズ
int[,] drawTip : 描画マップチップ
int[,] stateTip : 状態マップチップ
int px : キャラクタのx位置
int py : キャラクタのy位置

public Map(int arg_xSize, int arg_ySize) : テスト用コンストラクタ
処理 : 
xSize,ySizeにarg_xSize,arg_ySizeの値を代入
drawTip,stateTipをxSize,ySizeで初期化（以下のようにする）
drawTip = new int[ySize,xSize];	//y,xで指定することに注意！

public Update()
処理 :
Inputの内容を受け取って、方向キーによってpx,pyの値を変える（=キャラクタを移動する）
移動後px,pyの値が、0より少ないまたはxSize,ySize以上になるなら移動をキャンセル
移動先のsteteTipの値が1なら移動できないようにする（以下のようにする）
my = 移動後のy位置
mx = 移動後のx位置
if(stateTip[my,mx] == 1) {
	my = py;	//戻す
	mx = px;	//戻す
}
px = mx;
py = my;
少し無駄があるが、今後0,1でない値（たとえば毒沼=2とか？）が入る可能性「も」考えてこのようにする

public Draw()
処理 :
drawTipの値をすべて走査して、マップチップを表示する
2次配列の走査は以下のようにする。
for(int y = 0; y < ySize; y++){
	for(int x = 0; x < xSize; x++){
		//drawTip[y,x] とすると、順に値を見ていける
	}
}
drawTipの値が0なら、(x*32,y*32)から32×32の黒い四角形を描画する。
drawTipの値が1なら、(x*32,y*32)から32×32の白い四角形を描画する。
キャラクタの位置px,pyから、キャラクタを表す青い四角形を描画する。
(px*32,py*32)から32×32の青い四角形を描画すれば良い。


チェックする振る舞い : 
コンストラクタにテスト用のコードを追加する。
stateTipとdrawTipの適当な場所に1を挿入する。
stateTipとdrawTipが同じ内容にするほうがテストしやすい。
実際にインスタンスを生成、UpdateメソッドとDrawメソッドを呼び出す。
